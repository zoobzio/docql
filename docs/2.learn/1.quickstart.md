# Quickstart

## Requirements

Go 1.24 or later.

## Installation

```bash
go get github.com/zoobzio/docql
go get github.com/zoobzio/ddml
```

## Basic Usage

```go
package main

import (
    "fmt"
    "github.com/zoobzio/ddml"
    "github.com/zoobzio/docql"
    "github.com/zoobzio/docql/pkg/mongodb"
)

func main() {
    // 1. Define your schema
    schema := ddml.NewSchema("myapp").
        AddCollection(
            ddml.NewCollection("users").
                AddField(ddml.NewField("_id", ddml.TypeString)).
                AddField(ddml.NewField("username", ddml.TypeString)).
                AddField(ddml.NewField("email", ddml.TypeString)).
                AddField(ddml.NewField("active", ddml.TypeBool)),
        )

    // 2. Create a DOCQL instance
    instance, err := docql.NewFromDDML(schema)
    if err != nil {
        panic(err)
    }

    // 3. Build a query
    result, err := docql.Find(instance.C("users")).
        Filter(instance.Eq(instance.F("users", "active"), instance.P("is_active"))).
        Select(instance.F("users", "username"), instance.F("users", "email")).
        SortAsc(instance.F("users", "username")).
        Limit(10).
        Render(mongodb.New())

    if err != nil {
        panic(err)
    }

    fmt.Println(result.JSON)
    // {"collection":"users","filter":{"active":":is_active"},"projection":{"username":1,"email":1},"sort":[{"username":1}],"limit":10}

    fmt.Println(result.RequiredParams)
    // [is_active]
}
```

## What's Happening

1. `ddml.NewSchema` creates a schema definition
2. `ddml.NewCollection` and `AddField` define your collection structure
3. `docql.NewFromDDML` creates a validated instance bound to your schema
4. `instance.C`, `instance.F`, `instance.P` create validated references
5. `docql.Find` starts a query builder chain
6. `.Render(provider)` produces JSON and a list of required parameters

## Using with MongoDB Driver

The output is designed for use with the MongoDB Go driver:

```go
result, _ := docql.Find(instance.C("users")).
    Filter(instance.Eq(instance.F("users", "active"), instance.P("is_active"))).
    Render(mongodb.New())

// Parse the JSON and substitute parameters
var query bson.M
json.Unmarshal([]byte(result.JSON), &query)

// Execute with MongoDB driver
params := map[string]any{"is_active": true}
// Substitute :is_active with actual value
cursor, err := collection.Find(ctx, bson.M{"active": params["is_active"]})
```

## Multi-Provider Support

The same query builder works with different document databases:

```go
// Build query once
query := docql.Find(instance.C("users")).
    Filter(instance.Eq(instance.F("users", "active"), instance.P("active")))

// Render for MongoDB
mongoResult, _ := query.Render(mongodb.New())

// Render for CouchDB (Mango queries)
couchResult, _ := query.Render(couchdb.New())

// Render for Firestore
firestoreResult, _ := query.Render(firestore.New())

// Render for DynamoDB
dynamoResult, _ := query.Render(dynamodb.New())
```

Each provider produces output optimized for its target database.
